# EduGenie — Google Gemini Powered Learning Assistant

A lightweight AI-powered educational assistant built with **FastAPI** and a
responsive **HTML + CSS** frontend. EduGenie helps students understand
concepts, ask questions, generate quizzes, summarize material, and get
personalized learning paths — backed by **Google Gemini 1.5 Pro**, with an
optional local **LaMini-Flan-T5** fallback for offline / no-API-key use.

## Features

| Feature | Endpoint | Description |
|---|---|---|
| Intelligent Question Answering | `POST /api/ask` | Direct answers plus extra educational context |
| Simplified Concept Explanations | `POST /api/explain` | Explanations tuned to beginner / intermediate / advanced |
| AI-Powered Quiz Generation | `POST /api/quiz` | Multiple-choice quizzes with explanations |
| Educational Text Summarization | `POST /api/summarize` | Short / medium / detailed summaries |
| Personalized Learning Path | `POST /api/learning-path` | Staged roadmap with focus areas and time estimates |

## 1. Setup

**Requirements:** Python 3.10+, pip, a Google Gemini API key (free tier
available at https://aistudio.google.com/app/apikey).

```bash
# from inside the edugenie/ folder
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate

pip install -r requirements.txt
```

> The `transformers` + `torch` lines in `requirements.txt` are only needed
> for the local fallback model. If you always have a Gemini key, you can
> comment those two lines out to skip a ~3GB install.

## 2. Configure your API key

```bash
cp .env.example .env
```

Edit `.env` and paste your key:

```
GEMINI_API_KEY=your_real_key_here
```

If you leave `GEMINI_API_KEY` blank, EduGenie automatically falls back to
the local `LaMini-Flan-T5` model (downloaded automatically on first use).
Set `ENABLE_LOCAL_FALLBACK=false` in `.env` if you'd rather it fail loudly
than silently use the weaker local model.

## 3. Run it

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Open **http://localhost:8000** in your browser.

## 4. API quick reference

```bash
curl -X POST http://localhost:8000/api/ask \
  -H "Content-Type: application/json" \
  -d '{"question": "Which is the largest ocean?"}'

curl -X POST http://localhost:8000/api/quiz \
  -H "Content-Type: application/json" \
  -d '{"topic": "Pythagoras Theorem", "num_questions": 5, "difficulty": "medium"}'

curl -X POST http://localhost:8000/api/learning-path \
  -H "Content-Type: application/json" \
  -d '{"topic": "SQL", "current_level": "beginner", "goal": "be job-ready"}'
```

Interactive API docs are auto-generated by FastAPI at
**http://localhost:8000/docs**.

## Notes on architecture

- **Cloud model:** Google Gemini 1.5 Pro via the `google-generativeai` SDK,
  used for all five learning tasks.
- **Local fallback model:** `MBZUAI/LaMini-Flan-T5-783M` via HuggingFace
  `transformers`, loaded lazily only if/when it's actually needed — this
  keeps startup fast and lets the app run on resource-constrained machines
  (e.g. Mac M1) as described in the project brief.
- **Prompt engineering:** all prompts live in `app/prompts.py` so they can
  be reviewed, tuned, or evaluated independently of the request/response
  plumbing in `main.py`.
- **Structured outputs:** the quiz and learning-path endpoints instruct
  Gemini to return raw JSON, which `ai_service.extract_json` parses
  defensively (stripping markdown fences, recovering from stray text)
  before validating it against the corresponding Pydantic schema.

## Testing without a real Gemini key

Set `ENABLE_LOCAL_FALLBACK=true` and leave `GEMINI_API_KEY` blank — the app
will use the local model for every request. The UI shows a badge on every
response ("Answered by Gemini" / "Answered by local model") so it's always
clear which backend produced a given answer, and the status dot in the
header shows which mode you're in.

## Deployment

### Vercel (recommended, zero-config)

This repo is ready to deploy on Vercel as-is — `vercel.json` is included
and Vercel auto-detects the FastAPI `app` in `main.py`. Full step-by-step
instructions (including why the local fallback model is left out of the
production build) are in **`docs/DEPLOY_VERCEL.md`**.

Quick version:
1. Push this repo to GitHub.
2. Import it at https://vercel.com/new.
3. Set `GEMINI_API_KEY` (and `ENABLE_LOCAL_FALLBACK=false`) in the
   project's Environment Variables.
4. Deploy.

### Other hosts (Render, Railway, Fly.io, Google Cloud Run, etc.)

1. Containerize with a `Dockerfile` (`python:3.11-slim` base, `pip install
   -r requirements.txt`, `CMD ["uvicorn", "main:app", "--host", "0.0.0.0",
   "--port", "8000"]`).
2. Deploy to any container host and set `GEMINI_API_KEY` as a
   secret/environment variable there.
3. Point a custom domain at the service and you're done — no database or
   persistent storage is required for the current feature set.
